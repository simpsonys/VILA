# Voice Interaction Log Analyzer (VILA)

VILA는 복잡한 음성 인터랙션 로그를 발화(utterance) 단위로 신속하게 분석하고 시각화하기 위해 개발된 데스크톱 애플리케이션입니다. 로그 파일을 드래그·드롭하여 분석하는 **Batch 모드**와, 안드로이드 `sdb` 등과 연동하여 실시간으로 로그를 모니터링하고 분석하는 **Live (Stream) 모드**를 모두 지원합니다. 개발자와 QA 엔지니어가 디버깅 및 품질 분석에 소요되는 시간을 획기적으로 단축할 수 있도록 돕습니다.

---

## ✨ 주요 기능 (Features)

### 1. 강력한 로그 수집 및 분석 모드
- **Batch 모드 (드래그 앤 드롭)**: 기존에 저장된 로그 파일(`.log`, `.txt`)을 화면에 드래그하거나 클릭하여 열면 전체 내용을 자동으로 고속 파싱합니다.
- **Live 모드 (실시간 스트리밍)**: 단말(예: `sdb shell dlogutil`)과 직접 연결하여 실시간으로 쏟아지는 로그를 모니터링하고 즉시 발화 단위로 분석 결과를 제공합니다.
- **클립보드 붙여넣기 지원**: 텍스트 에디터나 웹 등에서 복사한 로그를 앱 내에서 `Ctrl+V`로 붙여넣기만 해도 바로 분석이 시작됩니다.
- **자동 인코딩 감지**: UTF-8, EUC-KR, UTF-16 등 다양한 파일 인코딩을 자동 감지하여 한글 깨짐 없이 안정적으로 처리합니다.

### 2. 사용자 맞춤형 분석 설정 (프리셋 지원)
- **프리셋(Preset) 관리**: 다양한 프로젝트나 타겟 로그 포맷에 맞춰 여러 개의 `pattern_config` 파일을 생성하고, UI 상단의 버튼으로 간편하게 전환하며 분석할 수 있습니다. (+ Custom Preset 추가 및 삭제 기능)
- **패턴 커스터마이징**: `JSON` 포맷의 설정 파일을 통해 로그 분석의 모든 규칙을 사용자가 직접 제어할 수 있습니다.
  - `start_patterns` / `end_patterns`: 발화 로그 블록의 시작과 끝을 구분.
  - `success_patterns` / `failure_patterns`: 발화의 성공/실패 여부를 판별.
  - `utterance_patterns`: 정규식을 통해 실제 사용자의 발화문을 추출.
  - `clickable_patterns`: `conversationId` 등 특정 ID를 추출하고 클릭 시 연동되는 하이퍼링크 URL 생성.
  - `table_columns`: 결과 테이블에 표시할 열(Column)을 자유롭게 구성.

### 3. 직관적인 UI 및 결과 테이블
- **사용자 조절 가능 결과 테이블**:
  - 각 칼럼의 헤더 경계선을 드래그하여 **너비(Width)를 자유롭게 리사이즈** 할 수 있습니다.
  - 칼럼별 개별 검색(Filter) 및 전체 통합 검색 지원.
  - 헤더 클릭을 통한 오름차순/내림차순 정렬 지원.
- **실시간 Live Log 창 내 검색 및 하이퍼링크**: Live 모드에서 제공되는 Raw 로그 창 내에서도 실시간 검색 및 검색어 하이라이팅을 지원하며, 패턴 설정에 따른 파란색 클릭 가능한 하이퍼링크가 그대로 적용됩니다.
- **TSV 및 복사 지원**: 현재 필터링된 결과만 TSV 형태로 클립보드에 복사하거나 별도 파일로 즉시 저장할 수 있습니다.

### 4. 깊이 있는 상세 분석 및 스크린샷 뷰어
- **발화별 상세 뷰 (Detail Window)**: 테이블의 특정 발화를 클릭하면, 해당 발화 블록과 관련된 **모든 유효 로그, 성공/실패 매칭 라인, 지정된 패턴 그룹**을 깔끔하게 분리된 UI로 조회할 수 있습니다.
- **자동 스크린샷 연동**: 분석 중인 로그 파일과 같은 위치에 있거나 설정된 `ScreenshotStorage`에 저장된 스크린샷이 있을 경우, 시점(Index/Utterance)에 맞춰 자동으로 매칭하여 상세 뷰에 함께 표시합니다.
- **스크린샷 뷰어 모달**: 상세 뷰에서 썸네일을 클릭하면 확대/축소(마우스 휠)가 가능한 뷰어 모달이 열립니다.
  - **📋 Copy**: 이미지를 클립보드로 바로 복사.
  - **💾 Save As**: 다른 이름으로 저장.
  - **📂 Reveal**: 탐색기에서 해당 파일 위치 바로 열기.

### 5. 완벽한 리포트 내보내기 (Export)
- **HTML + JSON 데이터 추출**: 분석된 결과를 타인과 손쉽게 공유할 수 있도록 **단일 HTML 파일 및 데이터 조각(JSON)** 들로 내보냅니다. VILA 앱이 설치되지 않은 사람도 웹 브라우저만 있으면 앱과 거의 동일한 상세 분석 및 테이블 필터링 기능을 경험할 수 있습니다.

### 6. 자동 업데이트
- 실행 시 백그라운드에서 GitHub 릴리스를 체크하여 최신 버전이 있을 경우 팝업을 띄우고, 원클릭으로 간편하게 업데이트 후 재시작합니다.

## 🚀 사용법 (Usage)

### 1. 기본 분석 시작하기
- 앱 실행 후 `Batch` 모드 상태에서 화면 중앙의 Drop Zone에 로그 파일을 끌어다 놓습니다.
- 또는 `⚡ Stream` 모드로 전환한 뒤 화면 하단의 SDB Device 주소를 확인하고 **🔴 Start Live Log** 버튼을 눌러 실시간 분석을 시작합니다. (단말 연결 필요)

### 2. 설정 (pattern_config.json) 수정하기
VILA의 가장 강력한 기능은 사용자가 직접 정규식(Regex)을 이용해 분석 규칙을 수정하는 것입니다.

1. 앱 상단 메뉴바의 **📂 Config** 버튼을 클릭하여 설정 파일이 위치한 사용자 데이터 폴더(Windows: `%APPDATA%\voice-log-analyzer`)를 엽니다.
2. 해당 폴더 내의 `default_config.json` 또는 `Preset0_...json` 파일을 텍스트 에디터로 엽니다. (수정 후 앱 재시작 필요)
3. UI에서 바로 `+ Custom Preset`을 눌러 기존의 JSON 파일을 불러와 새로운 프리셋으로 등록할 수도 있습니다.

## 🏗️ 빌드 및 배포

### 빌드 방법 (Windows exe)
1. **의존성 설치**: `npm install`
2. **개발 모드 실행**: `npm start`
   - *팁: Live Log 모드 UI 테스트를 위해 sdb 대신 `node mock-sdb.js` 명령어를 입력하고 Start를 누르면 가상 로그 스트리밍을 테스트해 볼 수 있습니다.*
3. **EXE 빌드**: `npm run build` (인스톨러), `npm run build:portable` (무설치 단일 실행 파일)
   - 빌드 결과물은 `dist/` 폴더에 생성됩니다.

### 배포 워크플로우
VILA는 소스 코드 저장소(`VILA`, Private)와 배포 저장소(`VILA_Release`, Public)를 분리하여 운영합니다.
1. `npm run build`로 생성된 `Setup.exe` 파일을 `VILA_Release` 저장소의 새 릴리스에 업로드합니다.
2. 앱에 내장된 자동 업데이트 로직이 해당 릴리스를 감지하고 다운로드를 수행합니다.
